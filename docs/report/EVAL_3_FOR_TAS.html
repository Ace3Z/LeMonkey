1. The task

Our robot (an SO-101 6-DOF arm) must place a coke can on the printed portrait of a celebrity named in a text prompt. Three printed celebrity portraits sit on the table; the prompt is, e.g., "Place the coke on Taylor Swift." The policy receives only the wrist camera, the text prompt, and proprioception.

Our recorded data is 180 human teleoperation episodes. We augmented these into ~10,000 episodes by digitally replacing the three printed portraits in each frame with new celebrity photos, in every permutation — so the policy sees many celebrities and many layouts. We built two augmented sets: one with 3 celebrities (Taylor Swift / Obama / Yann LeCun, our "TOY" set) and one with 200 celebrities.

2. The symptom — what works, what doesn't

The motor behaviour is solid. In every experiment below, the arm reliably picks up the can and places it down smoothly. The unsolved problem is identity: the policy does not reliably place the can on the correct celebrity. Placement is close to random with respect to the name in the prompt.

We trained on two vision-language-action (VLA) backbones: SmolVLA-450M (a SmolVLM2 vision-language model + an action expert) and Pi0.5-3B (a PaliGemma vision-language model + a Gemma-300M action expert).

3. Every experiment, in detail

SmolVLA experiments (reconstructed from team notes — approximate)

S1SmolVLA, image-as-prompt

Early attempt: we fed a reference photo of the target celebrity as a second camera input. Trained on ~4,200 episodes. On the robot it placed the can on the wrong celebrity (asked for Taylor Swift, placed on Obama). Abandoned once it was clarified that the policy may use only the scene camera + text + proprioception — no reference image.

S2SmolVLA, vanilla fine-tune

Stock SmolVLA fine-tuned on ~10,000 augmented episodes, action loss only. Places the can correctly but not on the queried celebrity.

S3SmolVLA with a face-warmed vision-language model

We first fine-tuned SmolVLA's vision-language model on ~200 celebrity images, then trained the action head on the ~10,000 episodes. Still does not select the correct celebrity. (This is a sequential setup — see the conclusion in §5, point 4.)

S4SmolVLA with an ArcFace auxiliary loss

SmolVLA trained on the 3-celebrity set (a 30,000-step run), with an added ArcFace-based distillation loss intended to inject face-identity structure into the vision features. The attention maps remained random; no improvement in identity selection. This experiment matters for the diagnosis in §4 — it shaped the face representations well and still didn't fix the behaviour.

Pi0.5 experiments (verified from the saved checkpoints)

P1Pi0.5, LoRA fine-tune on the TOY dataset

Data
9,394 episodes, 3-celebrity permutations (Taylor Swift / Obama / Yann LeCun).
Signal
The robot action loss only — nothing in this run ever told the model "this face is Obama."
Trained
LoRA adapters (small added weights) on the language model, the vision encoder's attention, and the action expert. The action input/output heads were left frozen.
Scale
30,000 steps, ~30 hours.
Result
Placement is smoother than any SmolVLA run, but it still does not select the correct celebrity.

Two recipe weaknesses we found afterward: the LoRA scaling factor came out about 8× weaker than we intended (a library default we failed to override), and the action heads were frozen rather than trained.

P2Pi0.5 face-recognition warm-start — language model only

Separately from the robot task, we took Pi0.5's vision-language half and fine-tuned it on a pure face-recognition task: the VGGFace2 dataset (~9,100 identities, ~3M images), question "Who is the person in this image?" → the person's name. In this version only the language model was adapted; the vision encoder stayed frozen.

Result: 37% on a 5-way face-identification test (random = 20%). A diagnostic probe explained the ceiling — with the vision encoder frozen, its features carry almost no identity information (same-person and different-person face pairs come out at nearly identical similarity, ~0.87 vs ~0.82 cosine). The language model cannot name someone from identity-blind features.

P3Pi0.5 face-recognition warm-start — vision encoder unfrozen

The same VGGFace2 face-recognition task, but this time we also adapted the vision encoder, so it can learn to encode who the person is, not just that a face is present.

Result: 48% on the 5-way face test (baseline 20%, previous version 37%) — a clear, monotonic improvement. On our 8 specific evaluation celebrities (8-way test), 2/8 correct, of which 1 (Taylor Swift) is a confident genuine hit. The weak score on our eval celebrities is expected: those particular people are not in VGGFace2, so the model is being asked to generalize to identities it never trained on.

Note: P3 is a model trained purely on face recognition. It has not yet been combined with the robot task — see §5, point 4, and §6.

4. The diagnosis — a controlled attention probe

Listing failures isn't a diagnosis. To find the mechanism, we ran a controlled, single-variable experiment.

The probe

Hold the scene completely fixed — the same three printed portraits, the same image. Vary only the celebrity name in the text prompt:

Swift  →  Obama  →  LeCun

Then inspect where the model's name token attends in the image. (In these models, the only way a word in the prompt can influence behaviour based on the image is by attending to the relevant image region — this is "cross-modal attention.")

Result

The name-token attention does not change as we vary the name. It parks on the same fixed background patch every time — an "attention sink" in a corner of the image — regardless of which celebrity is named. It never lands on the named celebrity's face.

Why this is the root cause

The task is "go to the named celebrity." The intended chain of reasoning inside the model is:

name token"Obama"
✗ →
cross-attentionroutes to Obama's face patch
→
VLM outputnow name-conditioned
→
action expertmoves to the right printout

The red link is the broken one. If the name-token attention is name-invariant, the name is never grounded to a face. The action expert downstream reads the vision-language model's output; with no name-conditioned difference in that output, the policy has no signal for which printout to move to — so it falls back to a positional guess. That single missing link is the entire task.

Why we are confident it is the cause, not a downstream symptom

EvidenceWhat it rules out
The probe isolates it. Same image, only the name varies — and the attention still doesn't move. Not a data, prompt, or scene-rendering problem. The one varied input fails to drive the one relevant mechanism.
Identical failure on every model. The same name-invariant attention appears on experiment S4, on P2, and on P3. Not a one-off checkpoint artifact. It is structural, across backbones and recipes.
Good representations did not fix it. Experiment S4's ArcFace loss shaped the face representations well — mean cosine 0.88 between matching face crops — and the attention routing still did not move. Rules out "the face features aren't good enough" as the gap. The faces are well-encoded; the name still cannot route to them.
Diagnosis

The gap is specifically attention routing — the name → face-patch binding — not representation quality, and not a downstream action problem. Every prior intervention improved an adjacent quantity and left this link untouched.

5. What we concluded

  1. The bottleneck is the vision encoder, not the language model. With a frozen vision encoder, the image features entering the language model don't encode who the person is — so adapting the language model alone cannot recover it (P2's probe is direct evidence).
  2. Action-loss-only training never teaches face matching. In every action-only run (S2, S3's second stage, P1), the vision features drift toward layout / position cues, not identity — hence the random placement.
  3. A face-recognition objective on the vision encoder does teach identity (37% → 48% as we unfroze the vision encoder, P2 → P3) — but teaching the model to recognize a face in isolation is not the same as teaching it to route a name to the right face in a 3-face scene.
  4. A sequential warm-start is not enough on its own. A face-trained model followed by an action-only robot fine-tune would lose the face skill during the second stage, because nothing in that stage keeps the identity ability graded.

6. What we are doing now

Two changes, both following directly from the diagnosis and from the co-training advice in the TA discussion.

Change 1 — Co-train, don't do it in stages

A single training run with two losses kept active simultaneously: the robot action loss on teleop episodes, and a face-identity loss on celebrity images. The face loss continuously holds the identity ability in place while the policy learns to act — it cannot drift away, because it is still being graded. We are including our specific eval celebrities, and face crops from our own printed-portrait footage, in the face stream — so the skill covers the exact identities and image conditions the evaluation uses.

Change 2 — Supervise the attention routing directly

Because the diagnosis pins the failure to attention routing specifically, a face-naming loss alone is not enough — naming a single isolated face never exercises the "which of three faces" decision. So we add an attention-supervision loss: for each training frame we know where the named celebrity's face is (a bounding box), we build a target attention pattern that puts weight on that face, and we directly penalize the model when its name-token attention diverges from that target. This is the only mechanism we have that addresses the diagnosed failure head-on rather than improving an adjacent quantity.

One important constraint we verified: this attention-supervision loss needs the vision encoder to be trainable — on a frozen, identity-blind encoder there is no discriminable face patch for the name to route to. With the encoder trainable, the attention-supervision gradient also drives the encoder toward identity-discriminable features. So the two changes reinforce each other: the face loss and attention supervision build the capability, the co-training keeps it alive while the policy learns to act.

Where we stand

The motor task is solved. The open problem — selecting the correct celebrity — has been narrowed by a controlled probe to one specific broken link: the name token never attends to the named face. Representation-quality fixes could not close it because they never trained routing. Our current run co-trains the action loss with a direct attention-supervision loss, on a trainable vision encoder, so the name → face binding is built and maintained in the same training that learns the robot behaviour.
