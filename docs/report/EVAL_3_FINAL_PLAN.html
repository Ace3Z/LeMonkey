owner: Roham backbone: lerobot/pi05_base brev: ~24 h on RTX PRO 6000 bonus: +16 repo output: HBOrtiz/pi05_eval3_track_B

Why this is the capacity hedge. Pi0.5 is 3B params (6.6× SmolVLA) and PaliGemma was pretrained on WebLI — a 10B image-text web-scrape corpus where celebrities are densely represented. Freezing the VLM (train_expert_only=True) preserves this prior intact while the action expert learns the manipulation skill. Critical config: we deviate from Pi0.5's default (train_expert_only=False, freeze_vision_encoder=False) precisely because that default is what Hans saw cause catastrophic forgetting on SmolVLM ("the distribution started shifting too much"). π₀.5-KI paper Fig. 6(b) confirms the same: 94% → 74% OOD without web co-training. We're using the frozen-VLM workaround.

Track D (optional, on standby) — M2 ArcFace cosine distillation toolkit

owner: Darius brev: 0 until activated activation: only if Track A or B Strix tests show face confusion

What this adds. An auxiliary training loss that distills buffalo_l ArcFace face embeddings into SmolLM2/PaliGemma's mid-LLM hidden state at the Backbone2Enc layer, forcing the VLM to develop face-discriminative features from the wrist-camera view alone. From BlindVLA ([arxiv 2510.25616](https://arxiv.org/abs/2510.25616) Eq. 9):

$$ L_{\text{align}} = -\frac{1}{k} \sum_{j=1}^{k} \cos\!\left(F.\text{normalize}\!\left(\text{MLP}(h_l)_j\right), F.\text{normalize}(z_j)\right), \quad \lambda = 0.2 $$

where $h_l$ is the VLM's hidden state at the chosen layer (SmolLM2 layer 7/8 of 16; PaliGemma layer 10/12), $z$ is the cached buffalo_l embedding of the target face, and the MLP is frozen after init (BlindVLA Table 6 — trainable projector becomes a shortcut). Inference-time impact: zero (the loss only adds a gradient signal during training; the cached embeddings are pre-computed offline).

Subtasks (Darius — Day 2)

  1. eval_3/aug/cache_arcface_embeddings.py — walk every variant directory, read augmentation.json["workspace_photos"][target_pid], compute the buffalo_l embedding for the target face, save as aug_cache_target_arcface.npy in each variant dir. ~2 h dev box.
  2. eval_3/aug/face_align_projector.py — frozen 3-layer MLP module: 
    class FaceAlignProjector(nn.Module):
    def __init__(self, in_dim: int, hidden: int = 2048, out_dim: int = 512):
        super().__init__()
        self.net = nn.Sequential(
            nn.LayerNorm(in_dim),
            nn.Linear(in_dim, hidden), nn.SiLU(), nn.Dropout(0.1),
            nn.Linear(hidden, hidden), nn.SiLU(), nn.Dropout(0.1),
            nn.Linear(hidden, out_dim),
        )
        for p in self.parameters():
            p.requires_grad = False  # M1 frozen projector
  3. Patch modeling files at Backbone2Enc — expose the hidden state at the chosen layer (SmolLM2 layer 8 by default), compute $L_{\text{align}}$, add $0.2 \cdot L_{\text{align}}$ to the total loss in forward(). ~4 h eng.

Hold for Day 3 decision. Apply only if A or B shows face-discrimination weakness in Day-2 Strix test.

§3 · Hans's warm-VLM recipe (so we can mirror on PaliGemma if needed)

Hans's exact recipe, reconstructed from his Slack messages (please confirm before Roham mirrors on PaliGemma):

  1. Backbone: HuggingFaceTB/SmolVLM2-500M-Video-Instruct, truncated to the first 16 transformer layers (matching SmolVLA's num_vlm_layers=16). This is the published SmolVLA recipe per arxiv 2506.01844 Table 8 (N=L/2 best speed/quality).
  2. Dataset: VGGFace2 (arxiv 1710.08092) — 9,131 identities, 3.31M images, curated from Google search for famous public figures. Swift/Obama/LeCun and all the OOD eval candidates (top-50 famous people per task spec) are essentially guaranteed to be in this set.
  3. VQA format: per-sample, draw uniformly from
    • "Who is in this photo?" → ""
    • "Is this ?" → "Yes" (50%) or "No, this is ." (50%)
  4. Training method: LoRA (PEFT), not full SFT. Hans observed that full SFT causes catastrophic forgetting — attention shifts to fine-tuned classes, generic objects ("blue bowl") lose attention. This matches π₀.5-KI's published 94% → 74% OOD degradation. LoRA preserves the base capabilities.
  5. Rank / target modules: Hans to confirm exact values. Standard LoRA-for-VLM defaults are r=16, alpha=32, target_modules=["q_proj","k_proj","v_proj","o_proj"] on the LLM transformer blocks.
  6. Output: after fine-tuning, merge the LoRA adapter into the base weights (model.merge_and_unload()) so the result is a standard HuggingFace model loadable by AutoModel.from_pretrained. This is what gets pushed to HansOrtiz/smolvlm2_celeb_warm.

If we need to mirror on PaliGemma in Day 3: same dataset, same VQA templates, same LoRA-not-SFT principle. Target modules on PaliGemma's Gemma blocks. Estimated ~10 h Brev given PaliGemma is ~7× larger.

§5 · Dataset preparation (Roham, tonight)

5.1 · Prompt-relabel pass

The 9,216 aug variants and 178 base teleops have prompts assigned per the original 75/15/10 mixture from the pre-ruling design (75% default + 15% ref-only + 10% counterfactual). The latter two buckets are invalid under text-only — they assumed a reference image. Re-label all sidecars to the default bucket only:

# eval_3/scripts/relabel_prompts_v3.py — relabel sidecar.json + augmentation.json
# to use the default-name prompt template, drop counterfactual + ref_only.
import json, random
from pathlib import Path

DEFAULT_PARAPHRASES = [
    "Place the coke on {name}.",
    "Put the coke on {name}.",
    "Place the can on the photo of {name}.",
    "Set the coke down on {name}'s picture.",
    "Put the can on {name}'s photo.",
]
SHORT_TO_FULL_NAME = {"swift": "Taylor Swift", "obama": "Barack Obama", "lecun": "Yann LeCun"}

for root in (Path("datasets/eval3"), Path("datasets/eval3_track3_aug")):
    for ep in root.iterdir():
        if not (ep / "reference.json").is_file():
            continue
        sc = json.loads((ep / "reference.json").read_text())
        full = SHORT_TO_FULL_NAME[sc["target_celeb"]]
        rng = random.Random(hash(ep.name) % (2**32))
        sc["prompt"] = rng.choice(DEFAULT_PARAPHRASES).format(name=full)
        sc["prompt_bucket"] = "default"
        (ep / "reference.json").write_text(json.dumps(sc, indent=2))

§6 · Strix testing protocol (Mahbod)

Every checkpoint that finishes training goes through identical paces. Don't deviate — comparability across tracks depends on protocol consistency.

  1. Pull the checkpoint: huggingface-cli download HBOrtiz/<track> --local-dir /tmp/<track>.
  2. Spin up the Strix laptop, plug in the SO-101 arm + wrist cam. Confirm camera1 stream is live + the same physical setup as recording-time.
  3. Place 3 paper printouts in any SOL/OSL/etc. layout. Don't rearrange between rollouts within a track — keep the setup fixed for fair comparison.
  4. 3 rollouts per checkpoint:
    1. TOY: place a Swift printout in 3 positions, prompt "Place the coke on Taylor Swift." — does the arm reach the right physical slot? Record success/fail per rollout.
    2. Held-out IID: same celebs but use the held-out heldout_03/04/05 photos (the ones we didn't print at recording time — they're in the photo bank, just print a fresh copy).
    3. OOD: rotate one of the printouts to a non-IID celeb (e.g. Federer, Bezos — pick from VGGFace2 top-50). Prompt their name. Record.
  5. Log to shared sheet: {track, rollout_type, celeb, layout, success(0/1), notes}.
  6. Capture video of any failure mode (positional shortcut, wrong celeb, timeout).
