Eval 3 — Final Plan

One co-training run · 3-track ablation · SmolVLA · 2026-05-20

A clean restart. Every previous attempt — vanilla fine-tunes, sequential VLM warm-starts, ArcFace representation distillation — failed for the same reason, and a controlled probe pinned down why. This plan does one thing: a single co-training run that fixes the diagnosed failure directly, evaluated as a 3-track ablation so we learn which ingredient actually carries the result. 1. The diagnosis — why everything failed

Full detail is in EVAL_3_FOR_TAS.html and the attention-routing diagnosis; the experiment record is in EVAL_3_RESULTS.md. In one paragraph:

Root cause

A controlled probe — fix the scene, vary only the celebrity name in the prompt — showed the vision-language model's name-token attention does not move. It parks on a fixed background patch regardless of the name. The name is never bound to a face, so the action expert has no name-conditioned signal and falls back to a positional guess. The failure is attention routing, not representation quality: representation-only fixes (ArcFace distillation, mean face-crop cosine ≈0.88) left the routing untouched.

Two consequences shape this plan. (a) The fix must supervise the routing itself, or run a loss that continuously exercises it — not just improve an adjacent quantity. (b) Sequential warm-starts fail because a face skill learned in stage 1 drifts away during an action-only stage 2; the identity signal has to stay graded the whole time. Hence: co-training.

2. The plan — one co-training run

A single training run with multiple losses active simultaneously, all sharing one SmolVLA backbone through the same LoRA adapters. This follows the co-training principle of π₀.₅ (arXiv 2504.16054) and Knowledge Insulation (arXiv 2505.23705), and the recipe of ObjectVLA (arXiv 2502.19250), which is the closest published method to our exact failure mode (see §3).

The loss components

  • $L_\text{action}$ — the normal SmolVLA flow-matching action loss on teleop episodes. Unchanged. Touches the whole policy (VLM + action expert).
  • $L_\text{vl}$ — a co-training loss on a stream of celebrity vision-language pairs: next-token prediction on bbox-grounded captions. ObjectVLA's recipe — the bounding box of the named face is encoded as text in the caption. Backprops into the VLM only. Mixed with the teleop stream at a 10:1 robot:VL ratio (ObjectVLA's verified ratio).
  • $L_\text{attn}$ (KLAL) — an attention-supervision loss. From the same bounding box we build a target attention map over the image patches and apply a KL-divergence loss pushing the name-token's attention toward the named face. This is the only loss that addresses the diagnosed routing failure head-on. Based on KLAL ("KL Attention Loss", arXiv 2511.12738, WACV 2026).
  • $L_\text{name}$ — a dedicated VQA naming loss: shown a face crop, predict the celebrity name. A separate celebrity-photo / face-crop stream, distinct from the bbox-caption stream above. VLM-only. Purpose: a direct, continuously-graded identity signal so the naming skill cannot drift.

The full objective (Track T3 below):

$$L \;=\; L_\text{action} \;+\; \alpha\,L_\text{vl} \;+\; \lambda\,L_\text{attn} \;+\; \beta\,L_\text{name}$$

summed and backpropagated once. The three lighter tracks switch terms off — see §3.

Loss weighting — do not sum raw

$L_\text{action}$ (flow-matching MSE), $L_\text{vl}$/$L_\text{name}$ (cross-entropy, ~1–10 nats) and $L_\text{attn}$ (KL over patches) live on very different scales. Normalize each to unit running scale before weighting. Starting points: $\alpha\!\approx\!0.1\text{–}0.3$, $\beta\!\approx\!0.1\text{–}0.3$, $\lambda$ warmed from 0 over the first ~10–20% of steps to $\approx\!0.05\text{–}0.2$. A hard KL target is the term most likely to destabilize training — introduce it gently and tune $\lambda$ in isolation (Track T2) before stacking T3. Log every loss and its gradient norm at the LoRA parameters separately.

3. The 3-track ablation

The tracks form a cumulative ladder — each adds exactly one mechanism to the one before. They run in parallel, one owner each.

TrackOwnerObjectiveAdds vs previousQuestion it answers
T1Darius $L_\text{action} + \alpha L_\text{vl}$ — (baseline) Does plain ObjectVLA-style co-training fix it on its own?
T2Mahbod $L_\text{action} + \alpha L_\text{vl} + \lambda L_\text{attn}$ + KLAL attention supervision Does directly supervising the routing add over T1?
T3Hans $L_\text{action} + \alpha L_\text{vl} + \lambda L_\text{attn} + \beta L_\text{name}$ + dedicated VQA naming loss Does a dedicated naming loss add over T2?

T0 — the floor — is already known. Vanilla fine-tunes with no co-training (SmolVLA "S2", Pi0.5 "P1" in EVAL_3_RESULTS.md) were already run and both placed the can near-randomly with respect to the prompt name — i.e. at about chance (≈⅓ on three portraits). We do not re-run that; ≈⅓ correct-portrait rate is the bar the three tracks must clear.

Honest reading of the ablation — it is additive, not isolating

Because T1's VL stream already carries the bounding box as text, the comparison T2 − T1 measures KLAL's value on top of a baseline that already has text-form grounding — not KLAL in isolation. Likewise T3 − T2 adds both a loss term and a new face-crop data stream. Read the ladder as "what is the marginal value of each added mechanism," and do not over-claim "KLAL alone does X." If a clean isolation of KLAL is wanted later, run a variant where T1's VL captions are bbox-free.

T1 — vanilla co-training Darius

Vanilla SmolVLA, 10:1 robot:VL co-training, no enhancements. The pure ObjectVLA recipe. Datasets: HBOrtiz/so101_eval3_track3_v3_baseline (teleop) + HBOrtiz/eval3_objectvla_vl_pairs (bbox-grounded VL pairs). This is the load-bearing baseline: ObjectVLA reports that this recipe takes OOD object-targeting from 19% → 64% success when the bounding-box grounding is present. If T1 already works, the more experimental tracks become optional.

T2 — + KLAL attention supervision Mahbod

T1's recipe plus the $L_\text{attn}$ KL-divergence attention-supervision loss. This is the rung that targets the diagnosed failure directly. It is also the most experimental: KLAL is a 2026 method validated on VQA/visual-grounding benchmarks — not, to our knowledge, inside a VLA/robot policy. Applying it here is a genuine extension. See §8 for the failure modes to watch.

T3 — + dedicated VQA naming loss Hans

T2's recipe plus $L_\text{name}$ — a separate celebrity-photo / face-crop VQA stream ("who is this?" → name). The full objective. Tests whether a direct, continuously-graded naming signal adds discrimination on top of the attention supervision.

4. Backbone — SmolVLA

All three tracks use SmolVLA (arXiv 2506.01844, Hugging Face) — a 450M VLA: a SmolVLM-2 vision-language backbone + a ~100M flow-matching action expert. Rationale: it is small (the smallest-model bonus), the prior SmolVLA failures were action-loss-only or sequential — co-training is an untried recipe on it — and the conceptual precedent RT-2 (arXiv 2307.15818) showed that VLAs can do emergent placement-onto-symbol and person-targeting tasks (the widely-circulated "move the Coke can to Taylor Swift" was a DeepMind demo of RT-2, not a benchmarked paper task — but it is the conceptual target of this work).

SmolVLA layer truncation — a structural fact that constrains the recipe

SmolVLA keeps only the first 16 layers of SmolVLM-2's language model and discards the upper half (verified in the SmolVLA paper: "we use only the first 16 layers of the LLM within the VLM"). LoRA can therefore only adapt those 16 layers. This is a real risk — if name→face routing in the full VLM lives in the discarded upper half, no amount of LoRA on the lower 16 recovers it. This must be checked before launch — see §7.

LoRA target modules

The plan's instinct is to LoRA the Q/K projections — since attention scores are $\text{softmax}(QK^\top)$, adapting Q/K changes where the model looks. That is half-right. But whether a correctly-attended face actually propagates to influence the action expert is decided by the V/O projections and the MLP. Q/K-only LoRA can produce "attention looks right, behaviour unchanged."

Recommendation: default to q_proj, k_proj, v_proj, o_proj — this matches the team's own previously-validated Pi0.5 LoRA recipe, and Q/K-only is an unjustified deviation from it. Treat the LoRA target set as an ablation knob: if compute allows, compare Q/K vs Q/K/V/O vs +MLP. Rank 16–32, applied to the 16 retained layers.

Fallback backbone: FLOWER / FlowerVLA (arXiv 2509.04996, 950M, flow-based) is the documented alternative if SmolVLA's 450M capacity or its layer truncation proves inadequate (§7, §8).

Deployment feasibility. SmolVLA-450M inference should comfortably fit the 16 GB Strix deploy GPU and the <20 s / rollout latency rule — but the prior Pi0.5 plan never actually ran its deployment probe (the largest unmeasured risk in EVAL_3_RESULTS.md §7). Run a VRAM + latency probe once any track produces a checkpoint; do not assume.

Keep the celebrity set small — TA guidance

Per Liam: there is no need to recognize thousands of identities — restrict to the most-famous handful. The 3-celebrity TOY set (Swift / Obama / LeCun — all extremely well-known) is the right scope; it makes the learning target easier and convergence more stable. Do not chase the 200- or 9k-celebrity sets for this plan.

Train / test scope — state this explicitly

The $L_\text{name}$ and $L_\text{vl}$ streams include the actual evaluation celebrities' identities. For this task that is acceptable — the eval is those specific known people, not generalization to unseen identities — but it must be stated, and the success claim scoped to "placing the can on known celebrities". Crucially: hold out the exact eval-day portrait images and camera footage from every training stream, even though the identities appear. Identities may be seen; the specific eval images must not. For development, curate a held-out validation set of celebrity portraits photographed under eval-like conditions but used in no training stream — the §9 correct-portrait metric is measured on it.

6. Implementation

This is an add-on to lerobot-train, not a rewrite. The pieces:

  • A second dataloader for the celebrity / bbox stream, interleaved with the teleop loader at the 10:1 ratio.
  • A forward path for the VL batches that stops at the VLM (no action expert) — these batches only produce $L_\text{vl}$, $L_\text{attn}$, $L_\text{name}$.
  • Switch the VLM attention implementation to eager so the attention weights are readable for the KL term ($L_\text{attn}$). Fused/SDPA attention does not expose the per-head weights.
  • Sum the losses (after scale-normalization, §2) and backprop once.
  • $L_\text{attn}$ target map: build a soft distribution over patches from the bbox (a smoothed map, not a hard 0/1 box) — a hard target is a harsher KL objective. The KLAL paper derives the target from the box centre-line, smoothed and normalized; a soft Gaussian over the box is an acceptable equivalent.
  • Start by supervising only the attention head(s) that actually do grounding, not all heads averaged — see §7/§8.
  • Train the action expert at full capacity. The prior Pi0.5 run (P1) under-powered it — LoRA-bottlenecked at low rank with the action I/O heads frozen (EVAL_3_RESULTS.md §P1). Do not repeat that: the SmolVLA action expert should be fully trained by $L_\text{action}$.

7. Pre-flight — the gating check before any track launches

Owner: Mahbod (holds the attention-probe tooling from the earlier Pi0.5 probes). T1, T2 and T3 all block on the Step-0 result.

Step 0 — verify routing capacity in the bottom 16 layers

Before committing compute to any track, run the attention probe on SmolVLA and confirm that name→face routing can exist within the retained 16 layers — i.e. identify which of those layers / heads carry (or could carry) cross-modal grounding. If the relevant routing in the full SmolVLM-2 lived only in the discarded upper half, the entire $L_\text{attn}$ approach on SmolVLA is built on sand and the team should switch to FLOWER (§4) or accept SmolVLA without $L_\text{attn}$. This check is cheap and it gates the whole plan. Do it first.

Step 0b — sweep the teleop data for the slot-R miscoding bug

EVAL_3_RESULTS.md §5.1 found 4 of 151 source episodes with miscoded slot-R supervision (the augmentation pipeline failed to swap a Swift portrait for LeCun), affecting ~260 of 9,216 variants — never swept across the full set. Bad labels poison both $L_\text{action}$ and any bbox-derived target. Before launch, sweep all 9,216 variants of so101_eval3_track3_v3_baseline for the orig_R=S, new_R=L pattern and remove or re-render the suspect variants.

Output of Step 0: the specific layer/head indices to supervise with $L_\text{attn}$. KLAL's published recipe averages over all layers and heads; the literature (arXiv 2503.06287) shows only a few heads do visual grounding, so supervising all of them injects noise. Localize, then supervise.

8. Risks & open questions

From an independent soundness review of this recipe (2026-05-20):

RiskSeverityMitigation
"Attention looks right, behaviour unchanged." The action expert reads VLM features via its own cross-attention, not the VLM's internal attention map. Forcing the internal map to match a bbox does not guarantee the features handed to the action expert encode the right identity. HIGH The success metric is the behaviour (correct-portrait reach rate), never the KL curve or attention-IoU alone. See §9.
Layer truncation. If name→face routing lived in SmolVLM-2's discarded upper half, LoRA on the lower 16 cannot recover it. HIGH (potential blocker) The §7 Step-0 check. If it fails → FLOWER backbone.
KLAL is experimental for VLA. Validated on VQA/grounding, never on a robot policy. Wrong layer/head choice supervises noise. HIGH Localize heads in Step 0; supervise only those; warm $\lambda$ from 0.
Q/K-only LoRA under-powered. May move attention without propagating the result. HIGH Default Q/K/V/O (§4); ablate target set if compute allows.
No distractor supervision. $L_\text{attn}$ pushes attention toward the named face but not explicitly away from the two distractor faces; the failure is "attends to all three equally." MED Use a target map that is near-zero on the distractor faces (contrastive by construction); consider an explicit negative term.
Catastrophic forgetting / action regression. Co-training could degrade $L_\text{action}$ quality. Full SmolVLM SFT previously caused forgetting on this project. MED Track action-task quality across the run; LoRA (not full SFT) limits this.
450M capacity. Likely fine if the problem is purely routing; but celebrity identity is long-tail and $L_\text{name}$ may be doing real representational work, not just anti-drift. MED If T3 underperforms on identity, FLOWER (950M) is the capacity fallback.
Loss-scale imbalance. Raw-summed losses on different scales let one term dominate the LoRA update. MED Scale-normalize; log per-loss gradient norms (§2).

9. Success metric & decision points

The metric that decides everything is the rollout — the rate at which the arm places the can on the correct celebrity's portrait, on held-out eval images. KL loss, attention-IoU and probe scores are diagnostic instruments, not the verdict — "attention looks right, behaviour unchanged" is the most probable silent failure. 

Step 0 pre-flight (§7) — routing capacity in bottom-16 layers?
   pass  -> launch T1, T2, T3 in parallel
   fail  -> switch backbone to FLOWER, or run T1/T3 without L_attn

After the parallel runs, on held-out rollouts:
   T1 already solves it          -> ship T1 (simplest, smallest risk)
   T2 > T1                       -> KLAL earns its place; ship T2
   T3 > T2                       -> the naming loss adds; ship T3
   none beats the T0 floor       -> escalate; FLOWER backbone; ask for compute
