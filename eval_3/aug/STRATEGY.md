# STRATEGY — Eval 3 identity-preserving inpainting augmentation

> **Status:** drafted 2026-05-10. Locks the technical recipe for turning ~144
> recorded SO-101 demos into ~1700 effective training episodes by inpainting
> the printed celebrity portraits with verified web-mined alternative photos
> of the same celebrity. Drives implementation under `eval_3/aug/`.

## 0 · Why this exists

Eval 3 grading is 9 rollouts (5.55 pts each) split into 3 IID-known + 3 IID-
held-out + 3 OOD tiers. The Phase 1 PaliGemma probe (run on thor 2026-05-09)
returned **0/14 TOY** and **0/6 OOD** — the VLM cannot retrieve names from
faces zero-shot. Path B (text-only fine-tune, freeze the VLM) is therefore
empirically dead. Path A (image-as-prompt + co-train) is mandatory.

Path A's success hinges on the policy generalising face identity across:

1. **Same printed photo / different camera frames** (same demo, different
   timesteps) — trivially handled by single-photo demos.
2. **Different photo of same person** (held-out IID tier, runs 4–6) — the
   policy must recognise Taylor Swift in a *new* photo, not just the trained
   one.
3. **Same photo / different lighting + angle** (any tier) — covered by
   recording variation.
4. **Unknown people altogether** (OOD tier, runs 7–9) — handled by image-
   as-prompt: the policy doesn't need name recall, only face-matching, and
   we co-train on VGGFace2 to keep that capability sharp.

**The inpainting augmentation buys layer 2 for cheap.** Per Lin et al.
(ICLR 2025) data scaling laws, diversity of (env × object) pairs matters far
more than demos-per-condition; ~10× per-demo augmentation by replacing the
printed portraits with new photos of the same celebrities is the highest-
leverage thing we can do without re-recording.

## 1 · Goal

Given a recorded demo at `<root>/<episode_dir>/`, produce N augmented
variants where:

- The 3 portrait regions in every video frame are replaced with **different
  photos of the same celebrities** that were on the table during recording.
- The action / state labels (parquet) are **byte-identical** to the original
  — the policy was placing the can on portrait position X; X still shows the
  same celebrity in the augmented variant, so the demo is still valid.
- The replacement looks **natural to the VLA**: no visible seam, color and
  exposure match the surrounding frame, identity is unambiguous (ArcFace
  cosine to a held-out reference > 0.4).
- The replacement is **identity-deterministic**: the new photo is chosen by
  us, not generated by a model — no diffusion-induced face drift.

Each variant gets its own `reference.json` sidecar pointing at a *different*
held-out reference photo of the target celebrity (so the image-as-prompt
input is also varied across variants; Interleave-VLA pattern).

## 2 · Architecture overview

```
INPUT
  episode_dir/videos/.../file-000.mp4    (640×480 @ 30 fps, ~600 frames)
  episode_dir/reference.json             (metadata: target_celeb, layout, ...)

STAGE 1 — celebrity photo bank (one-time, per dataset)
  Wikimedia API + icrawler Bing fallback → ≥30 photos / celeb
  filter: InsightFace ArcFace cosine > 0.4 vs Wikipedia headshot reference
  → datasets/eval3_celebs/web/<celeb>/<id>.jpg

STAGE 2 — segment 3 portraits across all frames (per episode)
  GroundingDINO("photo . portrait . paper card") on frame 0 → 3 boxes
  filter aspect ≈ 0.7 (148:210 A5), area ∈ [10k, 60k] px²
  SAM 2.1 hiera-L image predictor → 3 high-quality init masks
  SAM 2.1 hiera-L video predictor → propagate masks across all 600 frames
  cache: COCO RLE per portrait per frame (~5 MB / video)

STAGE 3 — mask → 4-corner quadrilateral (per frame, per portrait)
  cv2.findContours → largest contour
  cv2.minAreaRect → cv2.boxPoints → order TL/TR/BR/BL
  occlusion: where SAM obj_score < 0 or mask area < 0.5 × rolling median,
             linearly interpolate corners from neighbours

STAGE 4 — composite (per frame, per portrait, per variant)
  pick a NEW photo of the same celebrity from the bank (different from
       the portrait that was on the table during recording)
  cv2.getPerspectiveTransform(new_photo_corners → frame_corners)
  cv2.warpPerspective(new_photo, INTER_LANCZOS4)            # sharp warp
  cv2.GaussianBlur σ=0.8                                     # MTF match
  reinhard_lab(warped, src_frame, sample=outer_ring)         # WB / exposure
  cv2.erode(mask, 3×3)                                       # halo trim
  cv2.seamlessClone(NORMAL_CLONE)                            # seam blend

STAGE 5 — encode + sidecar + identity-verify (per variant)
  ffmpeg-python with NVENC h264 → augmented file-000.mp4
  write reference.json with new held-out photo path
  InsightFace ArcFace QA: cosine of inpainted face crop to ref photo > 0.4
       drop the variant if any frame falls below threshold

OUTPUT
  augmented_dir/<original_episode_name>__var<N>/
    videos/observation.images.camera1/chunk-000/file-000.mp4
    data/chunk-000/file-000.parquet  (hard-linked from original — labels unchanged)
    meta/   ...                       (ditto)
    reference.json                    (new variant-specific photo + metadata)
```

## 3 · The 5 stages in detail

### 3.1 · Photo bank construction

**Sources** (in order of preference):

1. **Wikimedia Commons API** — `https://en.wikipedia.org/wiki/Special:FilePath/<name>`
   redirects to a high-quality canonical image. Polite User-Agent required.
   Yield: 1–2 photos per celeb. Free, license-clean.
2. **icrawler Bing engine** — Python lib, breadth, ~30/min before throttle.
   Yield: 30+ per celeb. License: filter by `license_image=any-licence` in
   icrawler config.
3. **HuggingFace datasets fallback** — `ProgramComputer/VGGFace2-HQ` mirror
   for the IID three. Used only if Wikimedia + icrawler fail.

**Verification (per photo):**

- Detect face with `insightface.app.FaceAnalysis(name='buffalo_l')`.
- If 0 or >1 faces → reject (we want one clean head/shoulder shot).
- Compute ArcFace embedding; compare to the **Wikipedia primary headshot**
  (we assume Wikipedia is correct — bootstrap reference). Accept if
  `cosine ≥ 0.4`. Anything below is likely a mislabelled scrape.
- Save `<celeb>/<id>_<cosine>.jpg` (cosine in filename for debug).

**Target:** ≥30 verified photos per IID celeb (Swift, Obama, LeCun) plus
≥10 verified photos per OOD candidate (Federer, Merkel, Musk, Beyoncé,
Bezos, Trump, Ronaldo, Messi, Williams, Harris) for the supplementary
extra-celeb demos.

**Output:** `~/LeMonkey/datasets/eval3_celebs/web/<celeb>/*.jpg`

### 3.2 · Video segmentation with SAM 2.1

**Why SAM 2.1, not SAM 3:**

| | SAM 2.1 hiera-L | SAM 3 |
|---|---|---|
| FPS @ 640×480, A100 | ~40 | 5–6 ([benchmark](https://github.com/facebookresearch/sam3/issues/425)) |
| Text prompts | No (need GroundingDINO) | Yes native |
| Maturity | Stable | New, evolving |
| Our scenario | Rectangular rigid objects, single text-prompt at frame 0 | Same — no advantage from native text |

**Initial detection (frame 0 only):**

Use **GroundingDINO** with text prompt `"photo . portrait . paper card"`.
Filter the output boxes:

- aspect ratio in [0.55, 0.85] (148:210 = 0.704, ±20% slack)
- area in [10 000, 60 000] px² (loose for varying camera height)
- top 3 by confidence

Feed each surviving box to SAM 2.1 image predictor (`SAM2ImagePredictor`)
to get a clean mask. Validate: mask's `cv2.minAreaRect` aspect should still
be in the filter's range (rejects faces / shadows misclassified as portraits).

**Manual override:** if GroundingDINO fails for a given video (likely <5%),
fall back to a click-prompt mode: human clicks the centre of each portrait
in frame 0, SAM 2.1 image predictor takes the click as a positive point
prompt. Single-line CLI flag `--manual <ep_dir>` triggers this.

**Video propagation:**

```python
from sam2.build_sam import build_sam2_video_predictor

predictor = build_sam2_video_predictor(
    "configs/sam2.1/sam2.1_hiera_l.yaml",
    "/path/to/sam2.1_hiera_large.pt", device="cuda",
)
state = predictor.init_state(video_path=str(video_mp4))
for portrait_id, mask in enumerate(initial_masks):
    predictor.add_new_mask(state, frame_idx=0, obj_id=portrait_id, mask=mask)

per_frame_masks = {}  # frame_idx -> {portrait_id: (rle, score)}
for frame_idx, obj_ids, mask_logits in predictor.propagate_in_video(state):
    for oid, logit in zip(obj_ids, mask_logits):
        mask_bool = (logit > 0).cpu().numpy()
        score = float(logit.max().item())
        per_frame_masks.setdefault(frame_idx, {})[oid] = (encode_rle(mask_bool), score)
```

**Why memory-bank propagation handles occlusion:** the gripper covers a
portrait briefly (≤30 frames at 30 fps). SAM 2's memory bank stores object
features across recent frames; the occlusion head emits a low score during
the occlusion window but the object is recovered when it reappears. We do
not add fresh prompts mid-occlusion.

**Cache format:** one pickle (or `.npz`) per video named
`<episode_dir>/portrait_masks.pkl` with the structure above. Per-video
size ~2–5 MB. Re-augmenting with new photos never re-runs SAM — only
re-renders the homography.

**Throughput:** ~22 s per 600-frame video on RTX Pro 6000 (3 portraits
propagated jointly through one state). 144 videos in ~55 min.

### 3.3 · Mask → ordered 4-corner quadrilateral

Three corner-extraction methods were considered; for rigid rectangles the
clear winner is `cv2.minAreaRect` + `cv2.boxPoints`:

```python
mask = decode_rle(per_frame_masks[fi][pid][0])
contours, _ = cv2.findContours(mask.astype(np.uint8), cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
contour = max(contours, key=cv2.contourArea)
rect = cv2.minAreaRect(contour)            # (center, (w, h), angle)
corners = cv2.boxPoints(rect)              # 4 × 2 float32
corners = order_tl_tr_br_bl(corners)       # disambiguate
```

`approxPolyDP` is unreliable when the mask has notches from gripper
occlusion (returns 5 or 3 vertices). Use it only as a sanity check
("did the polygon collapse to a quadrilateral? if not, something's odd").

**Corner ordering** (`order_tl_tr_br_bl`):

```python
def order_tl_tr_br_bl(pts: np.ndarray) -> np.ndarray:
    """pts is 4×2. Returns array ordered top-left, top-right, bottom-right, bottom-left."""
    centroid = pts.mean(0)
    angles = np.arctan2(pts[:, 1] - centroid[1], pts[:, 0] - centroid[0])
    # quadrants (clockwise from top-left): top-left = -3π/4..-π/4, top-right = -π/4..π/4,
    # bottom-right = π/4..3π/4, bottom-left = 3π/4..π or -π..-3π/4
    quadrant = np.where(angles < -np.pi*3/4, 3,                          # bottom-left wraparound
                np.where(angles < -np.pi/4,   0,                          # top-left
                np.where(angles <  np.pi/4,   1,                          # top-right
                np.where(angles <  np.pi*3/4, 2, 3))))                    # bottom-right / bottom-left
    order = np.argsort(quadrant)
    return pts[order]
```

**Occlusion handling:**

- Per frame, record the SAM `obj_score` from the propagation output.
- Compute a rolling median of mask area across the previous 15 frames.
- If `score < 0` OR `area < 0.5 × rolling_median`, mark the frame "occluded".
- For occluded runs (consecutive occluded frames), linearly interpolate the
  4 corners between the last valid frame before and the first valid frame
  after the run. This is exact under constant-velocity wrist motion (which
  we have at 30 fps).

**Output:** `<episode_dir>/portrait_corners.json`:

```json
{
  "0": {                                   // portrait_id 0
    "0":   {"corners": [[x,y],[x,y],[x,y],[x,y]], "occluded": false, "score": 12.4},
    "1":   {...},
    ...
    "599": {...}
  },
  "1": { ... },
  "2": { ... }
}
```

### 3.4 · The "make it look natural" recipe (Recommended tier)

This is the actual hard part — eliminating the seam and matching local
exposure without changing identity. Three independent literature surveys
converged on the same recipe.

**Per-frame, per-portrait:**

```python
def replace_portrait(
    src_frame: np.ndarray,             # (H, W, 3) uint8
    mask: np.ndarray,                  # (H, W)    uint8 binary
    new_photo: np.ndarray,             # (h, w, 3) uint8 high-res replacement
    dst_corners: np.ndarray,           # (4, 2)    float32 in src_frame coords, TL/TR/BR/BL
) -> np.ndarray:
    H, W = src_frame.shape[:2]

    # 1. Lanczos warp: high-quality perspective transform of new_photo into mask region
    h, w = new_photo.shape[:2]
    src_corners = np.array([[0,0],[w-1,0],[w-1,h-1],[0,h-1]], dtype=np.float32)
    M = cv2.getPerspectiveTransform(src_corners, dst_corners.astype(np.float32))
    warped = cv2.warpPerspective(new_photo, M, (W, H), flags=cv2.INTER_LANCZOS4)

    # 2. MTF match: tiny Gaussian blur to mimic camera lens + Bayer demosaic
    warped = cv2.GaussianBlur(warped, (0, 0), sigmaX=0.8)

    # 3. Reinhard color transfer in Lab space, sampled from a 5-px ring
    #    just outside the mask (captures local WB / exposure / shadow tint)
    ring_dilated = cv2.dilate(mask, np.ones((11, 11), np.uint8))
    ring = cv2.subtract(ring_dilated, mask)            # outer ring only
    warped = reinhard_lab(warped, src_frame, sample_mask=ring)

    # 4. Erode mask 1–2 px to drop JPEG halos at the print edge
    mask_eroded = cv2.erode(mask, np.ones((3, 3), np.uint8))

    # 5. Poisson NORMAL_CLONE: gradient-domain blend
    ys, xs = np.where(mask_eroded > 0)
    if len(ys) == 0:
        return src_frame                                # mask vanished, skip
    center = (int(xs.mean()), int(ys.mean()))
    out = cv2.seamlessClone(warped, src_frame, mask_eroded * 255, center, cv2.NORMAL_CLONE)
    return out


def reinhard_lab(src: np.ndarray, ref: np.ndarray, sample_mask: np.ndarray) -> np.ndarray:
    """Match src's mean/std in Lab to those of ref, sampled where sample_mask > 0."""
    src_lab = cv2.cvtColor(src, cv2.COLOR_BGR2LAB).astype(np.float32)
    ref_lab = cv2.cvtColor(ref, cv2.COLOR_BGR2LAB).astype(np.float32)
    src_mean, src_std = src_lab.mean((0,1)), src_lab.std((0,1)) + 1e-6
    ref_pix = ref_lab[sample_mask > 0]
    ref_mean, ref_std = ref_pix.mean(0), ref_pix.std(0) + 1e-6
    out_lab = (src_lab - src_mean) * (ref_std / src_std) + ref_mean
    return cv2.cvtColor(np.clip(out_lab, 0, 255).astype(np.uint8), cv2.COLOR_LAB2BGR)
```

**Why each step matters:**

| Step | What it solves | Skip? |
|---|---|---|
| Lanczos warp | Sharpness preservation through the perspective transform | No |
| Gaussian σ=0.8 | The wrist-cam at 640×480 has a real MTF (lens + Bayer + JPEG); a fresh photo pasted in looks too crisp without it | No |
| Reinhard from outer ring | The ring captures exactly the local WB / exposure at the portrait's location; this is what makes "this photo lives on this table" believable | No |
| Mask erosion | The mask boundary often catches one row of cardstock-edge or JPEG-block artifact pixels — eroding 1 px discards them | No |
| Poisson NORMAL_CLONE | Gradient-domain blending: kills any residual seam by forcing color continuity at the boundary, while preserving the new photo's interior gradients (faces, hair) | No |

**Why NOT MIXED_CLONE:** picks the dominant gradient per pixel — would let
the *original* portrait's face bleed back through the new one. NORMAL_CLONE
copies *source* gradients only.

**Why NOT diffusion methods** (AnyDoor / Paint-by-Example / IMPRINT /
ObjectDrop): they preserve only *semantic* identity via DINOv2 tokens, not
*pixel* identity. They will subtly redraw the face, breaking our "different
photo of same person, action labels valid" invariant.

**Latency:** ~10 ms / frame on RTX Pro 6000. 600 frames × 144 videos × 10
variants = 864 000 frames → **~2.4 hours total** on a single GPU.

### 3.5 · Premium tier (optional)

Add **Harmonizer** (ZHKKKe, ECCV'22) as a final pass:

```python
import torch

```

Adds ~20 ms / frame on full-frame 640×480 (Harmonizer benchmarks at
56 fps @ 1080p). Total cost goes from ~2.4 h → ~7 h — still tractable.

Use cases for Premium:
- We see systematic global colour mismatch between augmented and original
  in held-out frames after the recommended tier.
- Lighting changes mid-episode (the SO-101 lab has fixed overhead, but if
  we record at home with window light, scene WB drifts).

If the recommended tier looks visually good on the 5 quick episodes, skip
Premium.

### 3.6 · Identity verification

Per variant, sample 5 frames spread across the timeline. For each:

1. Extract the inpainted portrait region (homography-warp back to a
   normalised 224×224 face crop).
2. Run InsightFace (`buffalo_l`) face detection + ArcFace embedding.
3. Compare against the embedding of the held-out reference photo.
4. Report the **min** cosine across the 5 sampled frames.

Reject the variant if min cosine < 0.4. ArcFace's standard verification
threshold for "same person, different photos" is ~0.4 (with some sources
quoting 0.35–0.45). 0.4 is conservative.

In practice we expect ≥95% of variants to pass — most failures are due to
extreme glare or motion blur on a specific frame, in which case the
mitigation is to retry with a different replacement photo.

## 6 · Validation strategy

Three quality gates before we trust an augmented variant:

### Gate 1 — segmentation overlay sanity (per video, 1 frame)

For frame 0, save an annotated PNG showing the original frame with the 3
masks drawn as semi-transparent coloured polygons. Manual eyeball: do the
quads sit cleanly on the printed portraits, or are they catching the table
/ shadows / fingertips?

Tooling: `eval_3/aug/dbg_overlay.py` (not in critical path).

### Gate 2 — composite spot-check (per variant, 3 frames)

For each augmented variant, save 3 frames (start / middle / end) as PNGs
and a side-by-side animated GIF (original vs augmented). Manual eyeball
on the first 5–10 variants per pipeline change.

### Gate 3 — ArcFace cosine threshold (per variant, 5 frames, automated)

The identity verification described in §3.6. Hard reject below 0.4.

If all 3 gates pass for the 5 quick episodes, we ship to the main 144-ep
run with confidence. If gate 1 fails → re-run with manual click-prompts
for the offending video. If gate 2 fails → tune the recipe (most common
failure: the Reinhard transfer over-corrects when the outer ring is in
deep shadow; mitigation: dilate the ring by 11 px instead of 5, OR fall
back to global Reinhard). If gate 3 fails → drop the variant; pick a
different replacement photo.

## 7 · Failure modes + mitigations

| Failure | Symptom | Mitigation |
|---|---|---|
| GroundingDINO misses a portrait | Frame-0 detection returns 0 / 1 / 2 boxes | Manual click-prompt fallback (`--manual`) for that video |
| SAM 2 loses a portrait mid-occlusion | `obj_score < 0` for >30 consecutive frames | Linear corner interpolation handles up to ~30 frames; longer occlusions → drop the variant or split the demo at the occlusion boundary |
| Replacement photo identity drifts | ArcFace cosine < 0.4 in any sampled frame | Drop variant, sample a different replacement; if happens systematically, increase ArcFace verification threshold during photo mining (0.4 → 0.45) |
| Visible seam after compositing | Hard line at mask boundary in overlay GIF | Tune Reinhard ring dilation (5 px → 11 px), increase mask erosion (3 px → 5 px), or upgrade to Premium tier (Harmonizer pass) |
| Wrong colour cast | Pasted photo looks "too warm" / "too cool" | Reinhard is per-channel Lab; if a channel is over-weighted, switch to MKL ([color-matcher repo](https://github.com/hahnec/color-matcher)) |
| Print emulation absent | Augmented portrait looks unnaturally crisp / no paper grain | Add a tiny Gaussian-noise overlay (σ ≈ 1.5/255 luma only) — defer until we observe "policy detects augmentation" failures |
| Replacement photo's pose conflicts with original | Original was a head-shoulder portrait; replacement is full-body. Homography squashes it. | Filter replacement photos by face-bbox-area / image-area > 0.15 during photo mining |

## 8 · Implementation plan

Files to write under `eval_3/aug/`, in execution order:

```
eval_3/aug/
├── STRATEGY.md                      ✓ this file
├── 1_mine_celeb_photos.py           Wikimedia + icrawler + ArcFace verifier
├── 2_segment_video.py               GroundingDINO + SAM 2.1 video propagator
├── 3_extract_corners.py             mask → ordered quadrilateral, occlusion interp
├── 4_inpaint_video.py               warp + Reinhard + Poisson + encode (Recommended tier)
├── 5_verify_identity.py             ArcFace QA on each variant
├── pipeline.py                      orchestrator: loop episodes × variants
└── dbg/                             debug visualizers (overlay, side-by-side GIF)
    ├── dbg_mask_overlay.py
    └── dbg_compare_gif.py
```

**Critical-path ordering:**

1. `1_mine_celeb_photos.py` — needed for `4_inpaint_video.py` to have
   replacement photos. Standalone; no dependency on the other modules.
2. `2_segment_video.py` — produces `portrait_masks.pkl` per episode.
3. `3_extract_corners.py` — consumes masks, produces `portrait_corners.json`.
4. `4_inpaint_video.py` — consumes corners + photos, produces augmented
   `.mp4` + sidecar.
5. `5_verify_identity.py` — consumes augmented `.mp4` + photos, accept/reject.
6. `pipeline.py` — wires 2 → 3 → 4 → 5 in a per-episode × per-variant loop.

**Smoke milestones:**

- M1 — `1_mine_celeb_photos.py` produces 30 verified Swift photos.
- M2 — `2_segment_video.py` produces correct masks on quick_swift_SOL_ep01.
  Verified by `dbg_mask_overlay.py` on frame 0.
- M3 — `3_extract_corners.py` produces stable corners across all 600 frames
  of ep01. Verified by plotting corner trajectories vs time (should be smooth).
- M4 — `4_inpaint_video.py` produces a single augmented variant of ep01
  with replacement photo X. Verified by `dbg_compare_gif.py` (visually
  indistinguishable except for the swapped face).
- M5 — `5_verify_identity.py` confirms the M4 variant's ArcFace cosine
  passes.
- M6 — `pipeline.py` produces 5 variants × 5 episodes = 25 augmented
  variants. End-to-end smoke test.

After M6 passes on the 5 quick episodes, we trust the pipeline and run it
on the main 144-ep collection (when it exists).
